JAX-WS 4.0 Service Tester

Service Information

This page provides information about the JAX-WS 4.0 web service running on Spring Boot 3.

Service Endpoints

WSDL: http://localhost:8080/ws/users?wsdl
Service URL: http://localhost:8080/ws/users

Available Operations

1. getUser

Retrieve a user by ID

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:user="http://example.com/user"> <soapenv:Header/> <soapenv:Body> <user:getUser> <userId>123</userId> </user:getUser> </soapenv:Body> </soapenv:Envelope>

2. createUser

Create a new user

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:user="http://example.com/user"> <soapenv:Header/> <soapenv:Body> <user:createUser> <user> <id>456</id> <name>Jane Smith</name> <email>jane.smith@example.com</email> </user> </user:createUser> </soapenv:Body> </soapenv:Envelope>

3. updateUser

Update an existing user

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:user="http://example.com/user"> <soapenv:Header/> <soapenv:Body> <user:updateUser> <user> <id>123</id> <name>John Doe Updated</name> <email>john.updated@example.com</email> </user> </user:updateUser> </soapenv:Body> </soapenv:Envelope>

4. deleteUser

Delete a user by ID

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:user="http://example.com/user"> <soapenv:Header/> <soapenv:Body> <user:deleteUser> <userId>123</userId> </user:deleteUser> </soapenv:Body> </soapenv:Envelope>

5. getAllUsers

Retrieve all users

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:user="http://example.com/user"> <soapenv:Header/> <soapenv:Body> <user:getAllUsers/> </soapenv:Body> </soapenv:Envelope>

Testing Tools

You can test the web service using any of these tools:

  • SoapUI - Professional SOAP testing tool
  • Postman - Modern API testing with SOAP support
  • cURL - Command-line HTTP client
  • Browser - Access the WSDL directly
Note: This service uses SOAP 1.2 with Document/Literal binding style. Make sure your SOAP client supports SOAP 1.2.

Service Configuration

  • SOAP Version: SOAP 1.2
  • Binding Style: Document/Literal
  • Namespace: http://example.com/user
  • Port: 8080
  • Context Path: /ws
